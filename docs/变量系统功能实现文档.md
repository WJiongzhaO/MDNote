# MDNote 变量系统功能实现文档

## 一、概述

### 1.1 系统定位

MDNote的变量系统是基于**模板处理器（TemplateProcessor）**的内容动态生成机制，允许用户在Markdown内容中使用变量占位符，在渲染时替换为实际值。该系统与项目的DDD（领域驱动设计）架构深度集成，遵循分层架构原则。

### 1.2 核心特性

- **Frontmatter变量支持**：在文档开头定义元数据变量，正文中可直接引用
- **文件夹级变量**：支持在文件夹中定义变量文件（`.mdnote-vars.yml`），子文档共享使用
- **变量继承机制**：文档变量优先级高于文件夹变量，支持多层级继承
- **变量替换**：支持 `{{变量名}}` 语法的简单变量引用
- **可视化编辑**：侧边栏UI用于管理文档和文件夹变量
- **实时预览**：在侧边栏中实时预览变量替换效果

### 1.3 应用场景

- **项目文档**：定义项目名称、版本号、作者等公共变量，在所有文档中引用
- **个人签名**：定义姓名、邮箱、个人域名等个人信息，快速插入文档
- **知识库管理**：在文件夹级别定义领域术语、通用链接等变量
- **快速替换**：修改变量值，所有引用自动更新（如批量修改版本号）

### 1.4 技术环境

**运行环境**：Electron 桌面应用
- ✅ 支持完整的文件系统访问
- ✅ 可直接读写文件夹中的 `.mdnote-vars.yml` 文件
- ✅ 支持多层级文件夹遍历

```
┌─────────────────────────────────────────────────────────────┐

│  - DocumentUseCases (文档用例)                             │
│  - FolderUseCases (文件夹用例)                             │
│  - VariableUseCases (变量用例 - 新增)                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Domain Layer (领域层)                                     │
│  - TemplateProcessor (模板处理器接口)                       │
│  - SimpleTemplateProcessor (简单替换实现)                  │
│  - FrontmatterParser (frontmatter解析器)                   │
```

```typescript
export interface TemplateProcessor {
  /**
   * 处理模板内容，替换变量
   * @param content 包含变量占位符的内容
   * @param variables 变量键值对
   * @returns 处理后的内容

  /**
   * 从内容中提取变量名列表
   * @param content 包含变量占位符的内容
   * @returns 变量名数组（去重）
   */
  extractVariables(content: string): string[];

  /**
   * 验证变量名是否合法
   * @param name 变量名
   * @returns 是否合法
   */
  validateVariableName(name: string): boolean;
}
```

#### 2.2.2 SimpleTemplateProcessor 实现类

**位置**：`src/domain/services/simple-template-processor.service.ts`

**职责**：实现简单的字符串替换模板处理

**核心功能**：
- 变量替换：`{{variable}}` → 替换为实际值
- 未定义变量：保持原样 `{{undefinedVar}}`
- 变量名验证：驼峰命名，只允许字母、数字、下划线

**实现方式**：使用正则表达式替换，无需引入额外模板引擎

```typescript
@injectable()
export class SimpleTemplateProcessor implements TemplateProcessor {
  private variablePattern = /\{\{([a-zA-Z_][a-zA-Z0-9_]*)\}\}/g;

  async processTemplate(
    content: string,
    variables: Record<string, any>
  ): Promise<string> {
    return content.replace(this.variablePattern, (match, varName) => {
      // 如果变量存在，替换；否则保持原样
      return variables.hasOwnProperty(varName)
        ? String(variables[varName])
        : match;
    });
  }

  extractVariables(content: string): string[] {
    const matches = content.matchAll(this.variablePattern);
    const variables = Array.from(matches, m => m[1]);
    return [...new Set(variables)]; // 去重
  }

  validateVariableName(name: string): boolean {
    return /^[a-zA-Z_][a-zA-Z0-9_]*$/.test(name);
  }
}
```

## 三、Frontmatter变量系统

### 3.1 什么是Frontmatter？

Frontmatter是一种在Markdown文件开头添加元数据的标准格式，使用YAML语法：

```markdown
---
title: "我的文档"
variables:
  projectName: MDNote
  version: 1.0.0
  author: 张三
tags: [笔记, 教程]
---

# {{projectName}} v{{version}}

作者：{{author}}
```

### 3.2 Frontmatter中的变量定义

#### 3.2.1 基本语法

```markdown
---
variables:
  name: value
  number: 42
  flag: true
  list:
    - item1
    - item2
---
```

#### 3.2.2 完整示例

```markdown
---
title: "项目文档"
variables:
  projectName: MDNote
  version: "1.0.0"
  author: "张三"
  email: "zhangsan@example.com"
  repoUrl: "https://github.com/example/mdnote"
  status: "开发中"
---

# {{projectName}} v{{version}}

**作者**: {{author}}
**邮箱**: {{email}}
**状态**: {{status}}

## 访问地址

项目主页: {{repoUrl}}

## 联系方式

如有问题，请发送邮件至 {{email}} 联系 {{author}}
```

### 4.1 文件夹变量文件

**文件名**：`.mdnote-vars.yml` 或 `.mdnote-vars.json`

**位置**：文件夹根目录

**示例**：
```yaml

```

### 4.2 文件夹变量继承

#### 4.2.1 继承规则

```
/project-root/.mdnote-vars.yml        { projectName: "MDNote" }
  /docs/.mdnote-vars.yml              { version: "1.0.0" }
    /api.md                           ← 可访问 projectName 和 version
  /examples/.mdnote-vars.yml          { mode: "demo" }
    /demo.md                          ← 可访问 projectName 和 mode
```

#### 4.2.2 继承优先级

```
查找顺序（从高到低）：
1. 文档frontmatter中的variables
2. 直接父文件夹的.mdnote-vars.yml
3. 祖父文件夹的.mdnote-vars.yml
4. 根文件夹的.mdnote-vars.yml
5. 全局配置（应用级）
```

## 五、变量查找优先级机制

### 5.1 完整的优先级链

```
1. 文档frontmatter.variables（最高优先级）
2. 直接父文件夹的.mdnote-vars.yml
3. 祖父文件夹的.mdnote-vars.yml
4. 根文件夹的.mdnote-vars.yml
5. 应用全局配置
```

### 5.2 变量合并策略

**位置**：`src/domain/services/variable-merger.service.ts`

```typescript
@injectable()
export class VariableMerger {
  private cache = new Map<string, Record<string, any>>();

  constructor(
    @inject(TYPES.FrontmatterParser) private frontmatterParser: FrontmatterParser,
    @inject(TYPES.FolderVariableResolver) private folderResolver: FolderVariableResolver
  ) {}

  /**
   * 合并所有来源的变量
   */
  async mergeVariables(context: {
    documentContent?: string;
    documentPath?: string;
    runtimeVariables?: Record<string, any>;
  }): Promise<Record<string, any>> {
    const cacheKey = this.getCacheKey(context);

    // 检查缓存
    if (this.cache.has(cacheKey)) {
      return this.cache.get(cacheKey)!;
    }

    const mergedVars: Record<string, any> = {};

    // 1. 应用全局配置
    Object.assign(mergedVars, this.getGlobalVariables());

    // 2. 文件夹变量（从低到高优先级）
    if (context.documentPath) {
      const folderVars = await this.folderResolver.resolveInheritedVariables(
        context.documentPath
      );
      Object.assign(mergedVars, folderVars);
    }

    // 3. 文档frontmatter变量（最高优先级）
    if (context.documentContent) {
      const frontmatterVars = this.frontmatterParser.extractVariables(context.documentContent);
      Object.assign(mergedVars, frontmatterVars);
    }

    // 4. 运行时变量（用于测试、预览等临时覆盖）
    if (context.runtimeVariables) {
      Object.assign(mergedVars, context.runtimeVariables);
    }

    // 缓存结果
    this.cache.set(cacheKey, mergedVars);

    return mergedVars;
  }

  /**
   * 清除缓存
   */
  clearCache(): void {
    this.cache.clear();
  }

  private getCacheKey(context: any): string {
    // 生成缓存键
    return JSON.stringify(context);
  }

  private getGlobalVariables(): Record<string, any> {
    // TODO: 从配置文件读取全局变量
    return {};
  }
}
```

### 6.3 应用层用例

**位置**：`src/application/usecases/variable.usecases.ts`

```typescript
@injectable()
export class VariableUseCases {
  constructor(
    @inject(TYPES.TemplateProcessor) private templateProcessor: TemplateProcessor,
    @inject(TYPES.FrontmatterParser) private frontmatterParser: FrontmatterParser,
    @inject(TYPES.FolderVariableResolver) private folderResolver: FolderVariableResolver,
    @inject(TYPES.VariableMerger) private variableMerger: VariableMerger
  ) {}

  /**
   * 渲染文档内容（替换变量）
   */
  async renderDocument(documentContent: string, documentPath: string): Promise<string> {
    // 1. 合并所有变量
    const variables = await this.variableMerger.mergeVariables({
      documentContent,
      documentPath
    });

    // 2. 处理模板
    return this.templateProcessor.processTemplate(documentContent, variables);
  }

  /**
   * 获取文档的所有变量（包括继承的）
   */
  async getDocumentVariables(
    documentContent: string,
    documentPath: string
  ): Promise<Record<string, any>> {
    return this.variableMerger.mergeVariables({
      documentContent,
      documentPath
    });
  }

  /**
   * 更新文档变量
   */
  updateDocumentVariable(
    content: string,
    key: string,
    value: any
  ): string {
    return this.frontmatterParser.updateVariables(content, { [key]: value });
  }

  /**
   * 获取文件夹变量
   */
  async getFolderVariables(folderPath: string): Promise<Record<string, any>> {
    return this.folderResolver.resolveFolderVariables(folderPath);
  }

  /**
   * 保存文件夹变量
   */
  async saveFolderVariables(
    folderPath: string,
    variables: Record<string, any>
  ): Promise<void> {
    await this.folderResolver.saveFolderVariables(folderPath, variables);
  }

  /**
   * 清除缓存
   */
  clearCache(): void {
    this.variableMerger.clearCache();
    this.folderResolver.clearCache();
  }
}
```

#### 7.1.2 组件实现

```vue
<script setup lang="ts">
import { ref, computed, watch } from 'vue';
import { useInject } from '@/core/inversify';
import { TYPES } from '@/core/container/types';
import type { VariableUseCases } from '@/application/usecases/variable.usecases';

// 通过依赖注入获取用例
const variableUseCases = useInject<VariableUseCases>(TYPES.VariableUseCases);

// 变量来源选择
const variableSource = ref<'document' | 'folder'>('document');

// 文档变量
const documentVariables = ref<Record<string, any>>({});

// 文件夹变量
const folderVariables = ref<Record<string, any>>({});

// 继承的变量（只读）
const inheritedVariables = ref<Record<string, any>>({});

// Props
const props = defineProps<{
  currentDocument?: { content: string; path: string };
}>();

// 加载文档变量
const loadDocumentVariables = () => {
  if (!props.currentDocument) return;

  const result = variableUseCases.frontmatterParser.parse(props.currentDocument.content);
  documentVariables.value = result.attributes.variables || {};
};

// 加载文件夹变量
const loadFolderVariables = async () => {
  if (!props.currentDocument) return;

  // 获取文件夹变量
  const folderPath = getFolderFromPath(props.currentDocument.path);
  folderVariables.value = await variableUseCases.getFolderVariables(folderPath);

  // 获取继承的变量（排除当前文件夹）
  const allFolderVars = await variableUseCases.folderResolver.resolveInheritedVariables(folderPath);
  const currentFolderVars = await variableUseCases.getFolderVariables(folderPath);

  // 计算继承的变量（从父文件夹）
  inheritedVariables.value = Object.keys(allFolderVars).reduce((acc, key) => {
    if (!(key in currentFolderVars)) {
      acc[key] = allFolderVars[key];
    }
    return acc;
  }, {} as Record<string, any>);
};

// 更新变量
const updateVariable = async (key: string, value: any) => {
  if (variableSource.value === 'document') {
    // 更新文档变量
    await updateDocumentVariable(key, value);
  } else {
    // 更新文件夹变量
    await updateFolderVariable(key, value);
  }
};

// 更新文档变量
const updateDocumentVariable = async (key: string, value: any) => {
  const updatedContent = variableUseCases.updateDocumentVariable(
    props.currentDocument.content,
    key,
    value
  );

  // 触发更新事件
  emit('update-document', updatedContent);

  // 重新加载
  loadDocumentVariables();
};

// 更新文件夹变量
const updateFolderVariable = async (key: string, value: any) => {
  const folderPath = getFolderFromPath(props.currentDocument.path);
  const updatedVars = { ...folderVariables.value, [key]: value };
  await variableUseCases.saveFolderVariables(folderPath, updatedVars);

  // 清除缓存
  variableUseCases.clearCache();

  loadFolderVariables();
};

// 添加新变量
const addVariable = () => {
  // 打开添加变量对话框
  editingVariable.value = { key: '', value: '' };
};

// 删除变量
const deleteVariable = async (key: string) => {
  if (variableSource.value === 'document') {
    const updatedContent = variableUseCases.frontmatterParser.removeVariable(
      props.currentDocument.content,
      key
    );
    emit('update-document', updatedContent);
    loadDocumentVariables();
  } else {
    const vars = { ...folderVariables.value };
    delete vars[key];
    const folderPath = getFolderFromPath(props.currentDocument.path);
    await variableUseCases.saveFolderVariables(folderPath, vars);

    variableUseCases.clearCache();
    loadFolderVariables();
  }
};
```

## 八、变量语法规范

### 8.1 变量引用语法

#### 8.1.1 基本变量引用

```
{{变量名}}
```

**示例**：
```markdown
# {{title}}

作者：{{author}}
日期：{{date}}

## 摘要
{{summary}}
```

**渲染结果**：
```markdown
# MDNote 项目文档

作者：张三
日期：2025-12-27

## 摘要
这是一个基于Markdown的笔记应用
```

#### 8.1.2 变量命名规范

- 使用**驼峰命名法**：`userName`, `createdAt`
- 使用**语义化名称**：`projectName` 而非 `n`
- 只能包含字母、数字和下划线
- 区分大小写：`UserName` 和 `userName` 是不同的变量
- 避免使用空格和特殊字符

**正确示例**：
```markdown
{{projectName}}
{{version}}
{{authorEmail}}
{{isPublished}}
```

**错误示例**：
```markdown
{{project-name}}     ❌ 不能使用连字符
{{project name}}     ❌ 不能包含空格
{{项目名称}}         ❌ 不能使用中文
{{user@email}}       ❌ 不能使用特殊字符
```

### 8.2 变量类型支持

#### 8.2.1 字符串类型

```markdown
---
variables:
  title: "MDNote"
  author: 张三
---

# {{title}} 作者：{{author}}
```

#### 8.2.2 数字类型

```markdown
---
variables:
  version: 1.0.0
  year: 2025
  count: 100
---

版本：v{{version}}
年份：{{year}}
数量：{{count}}
```

#### 8.2.3 布尔类型

```markdown
---
variables:
  isPublished: true
  hasWarning: false
---

状态：{{isPublished}}
```

**渲染结果**：
```
状态：true
```

#### 8.2.4 URL类型

```markdown
---
variables:
  repoUrl: "https://github.com/example/mdnote"
  website: "https://mdnote.dev"
---

项目地址：{{repoUrl}}
官方网站：{{website}}
```

## 十、常见问题

### 10.1 变量未替换？

**可能原因**：
1. 变量名拼写错误
2. 变量未定义
3. 变量名包含非法字符

**解决方案**：
```markdown
# 正确写法
{{projectName}}
{{version}}
{{authorEmail}}

# 错误写法
{{project-name}}     ❌ 使用了连字符
{{project name}}     ❌ 包含空格
{{项目名称}}         ❌ 使用了中文
```

### 10.2 如何查看当前文档有哪些变量？

**方法**：
1. 打开变量管理侧边栏
2. 选择"文档变量"或"文件夹变量"
3. 在"变量预览"区域查看所有可用变量

### 10.3 文档变量和文件夹变量冲突时怎么办？

**规则**：文档变量优先级高于文件夹变量

**示例**：
```yaml
# /project/.mdnote-vars.yml
version: "1.0.0"

# /project/doc.md (frontmatter)
variables:
  version: "2.0.0"

# 结果：文档中 {{version}} 显示为 "2.0.0"
```

### 10.4 如何在文档中使用文件夹变量？

**方法**：直接在文档中使用 `{{变量名}}` 引用即可

**示例**：
```markdown
# /project/.mdnote-vars.yml
projectName: "MDNote"

# /project/doc.md (无需定义frontmatter)
# 欢迎
{{projectName}}
```

### 10.5 变量值可以是多行文本吗？

**答**：目前不支持多行文本变量

**变通方案**：
```yaml
# 方案1：使用 \n 表示换行
description: "第一行\n第二行"

# 方案2：YAML 多行语法
longText: |
  这是一个很长的文本，
  可以分成多行。
```

### 10.6 如何批量修改变量值？

**场景**：修改项目版本号

**操作**：
1. 打开文件夹的 `.mdnote-vars.yml` 文件
2. 修改变量值
3. 保存后，所有引用该变量的文档自动更新

### 10.7 变量名区分大小写吗？

**答**：是

**示例**：
```yaml
variables:
  userName: "张三"
  username: "zhangsan"
  USERNAME: "ZHANGSAN"
# 这是三个不同的变量
```

### 10.8 如何删除不需要的变量？

**方法**：
1. 打开变量管理侧边栏
2. 选择变量来源（文档/文件夹）
3. 点击变量右侧的 ✕ 按钮
4. 保存即可

### 10.9 变量文件会破坏文档兼容性吗？

**答**：不会

Frontmatter是Markdown生态的标准格式，即使在没有MDNote的环境中：
- Frontmatter会被当作普通注释忽略
- 或者可以手动删除frontmatter部分
- 不影响Markdown内容的正常显示

### 10.10 如何导出/导入变量配置？

**方法**：
1. 文件夹变量：直接复制 `.mdnote-vars.yml` 文件
2. 文档变量：复制frontmatter部分到新文档
